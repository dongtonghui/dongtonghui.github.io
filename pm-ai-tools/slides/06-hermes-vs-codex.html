Hermes vs Codex

CODEX ← 对标

Codex = OpenAI 做的「专业码农」，
只写代码，只跑终端，只认 OpenAI 模型。

HERMES ← 我们的选择

Hermes = 开源的「全能管家」，
不限模型、不限平台、不限场景，
能学能记能进化。

维度 Codex Hermes
定位 专业编程助手，专注代码生成 全能 AI 管家，覆盖全场景
模型绑定 仅 OpenAI 模型 20+ 模型提供商，支持本地模型
平台覆盖 仅 CLI CLI / WebUI / Telegram / Discord / 钉钉 / 飞书 / API
核心能力 代码写作为主，终端操作为辅 代码 + 文档 + 分析 + 定时 + 多模态
记忆系统 无持久化记忆 内置记忆系统，跨会话知识积累
技能进化 依赖官方更新 用户自定义 Skill，自迭代演进
运行方式 本地 CLi 单次运行 7×24 云端服务，Cron 定时任务
开源 ❌ 闭源，受 OpenAI 服务限制 ✅ 完全开源，可私有化部署
