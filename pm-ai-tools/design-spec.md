# 设计 Spec：产品经理的AI工具使用分享

## 项目信息
- **主题**：面向产品经理的AI工具使用分享
- **场合**：内部团队分享
- **受众**：产品经理同事（熟悉产品工作流，对AI工具有基础认知但缺乏系统使用经验）
- **输出格式**：HTML幻灯片（1920×1080），浏览器演讲用
- **视觉风格**：科技极简风

## 核心信息

帮助产品经理理解：
1. 哪些工作环节可以被AI赋能（Reddit舆情分析、PRD撰写、Demo生成、每日复盘）
2. 如何选择合适的AI工具（全能性、多端在线、避免黑盒）
3. 为什么推荐Hermes（免费、skill丰富、多平台、自进化能力）
4. Hermes vs Codex的核心差异（全能管家 vs 专业码农）
5. 具体安装和最佳实践（Reddit分析、PRD撰写、文案review、每日复盘、skill进化）
6. 进阶技巧（苏格拉底式反问、女娲造人、沉淀skill）

## 内容结构（预计16-18页）

### 封面
- 标题：AI赋能产品经理工作流
- 副标题：从PRD到用户调研的效率革命
- 演讲者信息

### 第一章：哪些环节需要AI赋能
- Reddit舆情分析：指定版块+关键词+内容汇总
- PRD撰写：功能拆解、交叉评审、整合输出
- Demo生成：快速原型、交互验证
- 每日复盘：自动化日报、数据洞察

### 第二章：选择怎样的工具
- 选择依据：全能性、多端在线、避免黑盒
- 为什么是Hermes：免费、skill丰富、多平台、自进化
- Hermes vs Codex：一句话总结 + 详细对比表
- 选择技能的依据：能直接干活、能优化技能

### 第三章：Hermes架构优势
- 架构图：展示Hermes的多模型、多平台、skill系统
- 核心优势可视化

### 第四章：如何安装
- 本地安装：CLI + WebUI
- 云端安装：阿里云 + 宝塔Docker

### 第五章：最佳实践
- Reddit舆情分析（reddit-community-research）
- PRD撰写（prd-writing）
- 页面文案review（copy-review + darwin）
- 每日复盘
- 技能进化（darwin2.0）

### 第六章：其它技巧
- 苏格拉底式反问
- 女娲造人（nuwa-skill）
- 沉淀skill为工作流

### 附：Hermes vs Codex 详细对比
- 定位、模型绑定、平台覆盖、核心能力、记忆系统、技能进化、运行方式、安全模式、开源

### 技能使用差异
- Codex Extensions = 给工人换电钻
- Hermes Skills = 老师傅的装修笔记

### 封底
- 总结 + 行动号召

## 情感基调与气质关键词
- 专业但不冰冷
- 实用导向
- 科技感的信任感
- 内部分享的亲和感

## 视觉系统

### 色彩
- 主背景：纯白 #FFFFFF 或 极浅灰 #F8F9FA
- 主文字：深黑 #0A0A0A
- 强调色：科技蓝 #2563EB 或 电光蓝 #2D5BFF
- 辅助色：中性灰 #6B7280
- 点缀色：成功绿 #10B981（用于优势/正确标识）

### 字体
- 中文标题：思源黑体 / Noto Sans SC（粗体）
- 中文正文：思源黑体 / Noto Sans SC（常规）
- 英文标题：Inter / Geist（粗体）
- 英文正文：Inter / Geist
- 代码/数据：JetBrains Mono / Geist Mono

### 布局节奏
- 封面：大字标题 + 极简信息
- 章节页：大色块 + 章节号
- 内容页：顶部标题 + 结构化内容（列表/卡片/对比表）
- 数据页：大数字 + 简短说明
- 对比页：左右分栏或表格
- 封底：总结 + CTA

### 视觉主角轮换
- 封面：大字排版
- 架构页：系统架构图（HTML原生绘制）
- 对比页：表格/双栏对比
- 实践页：步骤卡片/流程图
- 技巧页：引用/要点高亮

## 图片需求
- Hermes logo（SVG或PNG）
- 架构图（HTML原生绘制，不需要外部图片）
- 对比表格（纯HTML/CSS）
- 流程图（纯HTML/CSS/SVG）

## 已知约束
- 无品牌色规范，使用科技蓝作为统一强调色
- 无特定logo资产，Hermes logo从官方渠道获取
- 内部分享，信息密度适中，避免过于学术化
- 中文为主，英文术语保留原文
